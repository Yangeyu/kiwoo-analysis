# Feature Specification: Manus-Style Canvas Analysis Workflow

**Feature Branch**: `001-manus-analysis-workflow`  
**Created**: 2026-02-27  
**Status**: Draft  
**Input**: User description: "创建一个类似 Manus 的工作流，对 Kiwoo 画布（boards、nodes、edges、viewport 及多类型节点 metadata）进行多维度、可扩展分析，并输出 Markdown 结构化报告。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 生成可读分析报告 (Priority: P1)

作为分析人员，我可以基于单个白板的数据快照发起一次分析，并获得一份结构清晰的
Markdown 报告，以便快速理解白板中的关键内容、结构关系和主要结论。

**Why this priority**: 这是核心交付价值，没有可读报告就无法支持后续讨论与决策。

**Independent Test**: 提供一份包含节点、连线和视口信息的白板快照，执行分析后可直接获得
结构完整、可阅读的 Markdown 报告，且报告包含结论与证据引用。

**Acceptance Scenarios**:

1. **Given** 白板快照包含 note、section、link 等常见节点，**When** 用户发起分析，
   **Then** 系统生成包含“概览、结构分析、重点发现、风险与建议”的 Markdown 报告。
2. **Given** 白板中存在 ai_note 且包含 referenced 信息，**When** 生成报告，
   **Then** 报告中的关键结论可追溯到对应引用来源。

---

### User Story 2 - 按分析维度扩展报告 (Priority: P2)

作为高级用户，我可以选择或组合不同分析维度（例如信息密度、关联强度、主题聚合、
内容可信度），以便输出更贴合目标场景的定制化报告。

**Why this priority**: 用户场景差异大，固定模板难以长期满足业务演进，需要可扩展分析维度。

**Independent Test**: 在相同白板快照上分别启用不同维度组合，结果报告在结构一致的前提下
体现出维度差异，并明确标记每个维度的结论。

**Acceptance Scenarios**:

1. **Given** 用户选择“结构拓扑 + 内容质量”两个维度，**When** 生成报告，
   **Then** 报告包含这两个维度的独立小节和综合结论。

---

### User Story 3 - 保持报告一致性与可复核性 (Priority: P3)

作为团队协作者，我希望多次分析输出保持统一格式，并且可对比不同时间点的分析结果，
以支持评审、复盘和协作沟通。

**Why this priority**: 一致格式和可复核性决定报告能否在团队中沉淀为可复用资产。

**Independent Test**: 对同一白板在不同时间点执行分析，报告版式保持一致，且能清晰显示
新增、变化和缺失的关键发现。

**Acceptance Scenarios**:

1. **Given** 同一白板有两次分析结果，**When** 用户查看对比输出，
   **Then** 可识别主要变化项及其影响说明。

---

### Edge Cases

- 白板为空或仅含极少节点时，报告仍需生成最小可读结果并说明信息不足。
- 节点类型混杂且部分 metadata 缺失时，系统需标注不确定性并继续完成可用分析。
- 连线存在断链、孤立节点或循环引用时，报告需明确拓扑异常并给出解释。
- 输入快照规模显著增大时，系统需保证输出完整性并提示分析粒度策略。
- 白板标题、内容文本或链接信息包含多语言时，报告需保持原文可追溯并避免语义丢失。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 接收单个白板数据快照作为分析输入，并校验其基础结构完整性。
- **FR-002**: 系统 MUST 解析节点、连线、视口及节点扩展字段，并形成统一分析上下文。
- **FR-003**: 用户 MUST 能选择一个或多个分析维度来控制报告生成范围。
- **FR-004**: 系统 MUST 产出结构化 Markdown 报告，至少包含概览、维度分析、关键发现、风险与建议，并根据上下文确定输出语言。
- **FR-005**: 系统 MUST 在报告中提供关键结论的证据引用，支持追溯到节点或引用片段。
- **FR-006**: 系统 MUST 对不同节点类型（note、ai_note、scribble、section、link、report、table）执行差异化分析规则。
- **FR-007**: 系统 MUST 对输入中的异常数据给出可读告警，并在不影响整体可用性的前提下继续生成报告。
- **FR-008**: 系统 MUST 支持同一白板的多次分析结果对比，输出变化摘要。
- **FR-009**: 系统 MUST 保证报告章节顺序与命名在同类任务中保持一致。
- **FR-010**: 系统 MUST 在报告中区分“事实观察”“推断结论”“待验证假设”，避免混淆。
- **FR-011**: 系统 MUST 按优先级解析报告语言（显式语言参数 > 用户上下文 > 白板内容语言特征 > 系统默认语言），并在结果中返回最终语言。

### Quality & Testing Requirements *(mandatory)*

- **QR-001**: 每项功能需求 MUST 有对应的可重复验证用例，覆盖正常、边界与异常输入。
- **QR-002**: 报告结构一致性 MUST 通过自动化校验，确保必需章节不缺失。
- **QR-003**: 结论可追溯性 MUST 可验证，即关键结论都能定位到至少一个输入证据。
- **QR-004**: 回归测试 MUST 覆盖主要节点类型与混合场景，确保扩展维度不破坏既有结论质量。

### User Experience Consistency Requirements *(mandatory for user-facing changes)*

- **UX-001**: 报告标题、章节命名与摘要语气 MUST 在同一目标语言下保持一致风格。
- **UX-002**: 报告中的风险、建议、结论展示 MUST 使用统一层级和术语，降低阅读负担。
- **UX-003**: 对异常输入、低置信度结论和缺失数据的提示 MUST 清晰、可理解且不误导。

### Performance Requirements *(mandatory)*

- **PR-001**: 对标准规模白板，90% 的分析任务应在 60 秒内返回完整报告。
- **PR-002**: 对较大规模白板，系统 MUST 在 3 分钟内完成报告，或明确返回分阶段结果与进度说明。
- **PR-003**: 报告生成成功率 MUST 在稳定输入条件下达到 99% 以上（按 7 日滚动窗口统计）。

### Key Entities *(include if feature involves data)*

- **Board Snapshot**: 单次分析输入对象，包含白板元信息、节点集合、连线集合和视口状态。
- **Whiteboard Node**: 白板节点实体，含公共字段与类型特定扩展数据，用于语义与结构分析。
- **Analysis Dimension**: 分析维度定义，描述某类分析目标、输入要求与输出章节规则。
- **Evidence Reference**: 证据引用实体，标记报告结论与源数据之间的可追溯映射关系。
- **Analysis Report**: 最终输出对象，包含结构化章节、关键发现、风险建议与变化摘要。
- **Comparison Result**: 多次分析的对比对象，记录新增、删除、变化项及影响说明。

### Dependencies

- 白板快照来源在分析发起时可访问，且具备最小字段完整性。
- 用户具备读取目标白板内容并生成分析报告的业务权限。
- 组织内存在统一的报告评审流程，用于验证结论可用性与行动项落地。

### Assumptions

- 默认输入已由上游提供为可读取的白板快照，且包含必要的基础字段。
- 默认分析对象为单白板；跨白板聚合分析不在本期范围。
- 默认报告语言由上下文自动推断，未提供有效上下文时回退到系统默认语言。
- 默认允许用户按需选择分析维度，但不要求用户自定义维度算法细节。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 95% 的目标用户可在 5 分钟内读完报告并复述至少 3 条关键发现。
- **SC-002**: 至少 90% 的关键结论具备明确证据引用且可被复核通过。
- **SC-003**: 在包含 7 种节点类型的混合样本中，报告完整输出率达到 99%。
- **SC-004**: 用户对“报告结构清晰度与一致性”的满意度达到 4.3/5 及以上。
- **SC-005**: 在连续两次白板变更后，变化摘要识别准确率达到 90% 及以上。
- **SC-006**: 团队评审中，基于报告直接形成可执行行动项的会议占比达到 80% 及以上。

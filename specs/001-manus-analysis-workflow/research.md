# Phase 0 Research: Manus-Style Canvas Analysis Workflow

## 研究主题 1：LangGraph 编排架构

**Decision**

- 采用 LangGraph + 类型化状态机 + 可插拔分析子图，实现分析工作流编排。

**Rationale**

- 工作流步骤显式（解析 -> 分析 -> 证据归集 -> 报告生成 -> 对比），便于追踪与调试。
- 节点级重试和条件分支可直接表达失败恢复与降级路径。
- 每个节点可输出标准化证据结构，天然支持可追溯要求。
- 新增分析维度时可注册子图，不需要重写主流程。

**Alternatives considered**

- LangChain Agent（无图编排）：上手快，但复杂流程可观测性和恢复能力较弱。
- Airflow/Prefect + LLM 调用：调度成熟，但对 LLM 上下文编排不如 LangGraph 原生。
- 自研编排引擎：可定制但成本与风险高，不适合当前迭代节奏。

## 研究主题 2：上下文语言 Markdown 报告质量控制

**Decision**

- 采用“双层模板 + 规则校验 + 轻量评审”机制保障上下文语言报告质量与一致性。

**Rationale**

- 固定章节模板可避免漏段、串段与结构漂移。
- 术语字典校验可确保同义词收敛，降低阅读歧义。
- 强制区分事实与推断，并要求推断引用事实 ID，可降低幻觉风险。
- 低置信度项必须标注原因和建议动作，便于业务侧审慎决策。

**Alternatives considered**

- 仅靠 Prompt：稳定性不足，难保证引用闭环。
- 全人工审稿：质量可控但吞吐低、成本高。
- 端到端模型微调：更新慢且仍需要规则层兜底。

## 研究主题 3：白板图数据分析策略

**Decision**

- 采用“属性图 + 多视图（结构图/语义图）+ 分层分析管道”的分析方案。

**Rationale**

- 结构关系与语义关系拆分后，拓扑指标更稳定、解释更清晰。
- 分层管道（标准化 -> 构图 -> 特征 -> 检测 -> 输出）便于扩展维度。
- 异常检测结合规则与统计可兼顾高精度与高召回。
- 增量计算 + 预计算 + 缓存策略可满足大规模输入下的时延目标。

**Alternatives considered**

- 纯关系型分析：路径/社区/连通等问题表达能力不足。
- 单一大图：语义边与结构边混杂导致指标污染。
- 纯实时或纯离线：分别在成本或时效上失衡。

## 研究主题 4：测试与性能门禁

**Decision**

- 建立 Commit/PR/Main/Release 四级门禁，绑定代码质量、测试、UX 一致性、性能预算。

**Rationale**

- 分层测试可快速定位问题并覆盖结构回归与语义回归。
- 将性能预算硬门禁化可防止“慢性退化”进入主干。
- 增加报告快照对比与结论漂移告警，可提升输出稳定性。
- 发布前验证包可统一执行可追溯性与异常场景检查。

**Alternatives considered**

- 仅端到端测试：定位慢、盲区大。
- 无性能阈值：长期易积累性能债。
- 仅人工 UX 评审：主观性高且不可规模化。

## 结论汇总

- 所有 Technical Context 决策已明确，无未解决的 NEEDS CLARIFICATION。
- 编排框架确认：LangGraph。
- 输出语言确认：由上下文解析（含术语一致性与低置信度标注规范）。
- 进入 Phase 1 设计与契约定义。
